- 시스템 안정성 확보로 서비스 중단 없이 운영
- k8s 기반 자동 스케일링으로 운영 효율성 증대
- 지속적 모델 학습 환경(CI/CD/CT) 구축으로 예측 정확도 유지

- 전체 시스템 아키텍처 설계 및 성능 최적화 전략 수립 (기여도: 80%)
- FastAPI 기반 백엔드 API 개발 및 ML 모델 서빙 인터페이스 구현 (기여도: 70%)
- 대용량 트래픽 대응을 위한 인프라 설계 및 구축 (기여도: 70%)
- 5명 개발자(백엔드, 프론트엔드, MLOps)로 구성된 팀 리딩

아키텍처 설계:
1. 마이크로서비스 구조: 기능별로 3개 서비스(모델 서빙, 지리정보 연산, 가게 정보관리)로 분리하여 독립적 확장 가능한 아키텍처 구축
2. 트래픽 처리 최적화: Python/FastAPI의 비동기 기능을 활용한 I/O 병목현상 최소화
3. 다중 레벨 캐싱: Redis 클러스터 기반 캐싱으로 반복 요청 성능 개선
4. 사전 스케일링: CronJob을 통해 피크타임 2시간 전 자동으로 Node와 Pod 확장
5. 컨테이너 최적화: 서비스별 특성에 맞는 리소스 할당 및 Probe 설정 최적화

이러한 설계는 Python의 GIL(Global Interpreter Lock) 한계를 극복하고 대규모 트래픽에 효과적으로 대응하기 위한 전략이었습니다. 특히 파이썬 서비스의 성능 최적화를 위해 각 서비스의 특성을 분석하고, k8s 설정(Pod별 리소스 할당, Probe 설정, HPA 설정)을 정밀하게 조정했습니다.

실제 성과:
- 평균 응답시간을 100ms에서 20ms로 80% 개선
- 피크타임 성능 안정화로 사용자 경험 크게 향상
- 시스템 안정성 확보로 서비스 중단 없이 운영
- k8s 기반 자동 스케일링으로 운영 효율성 증대
- 지속적 모델 학습 환경(CI/CD/CT) 구축으로 예측 정확도 유지
